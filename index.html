Github 上部署Hexo个人博客的过程

Posted on 2019-11-02

Github 上部署Hexo个人博客的过程

创建GItHub域名和空间

首先需要注册一个github账号，其中username需要认真注意，毕竟最终的个人博客的域名将会是username.github.io。

创建仓库

然后需要创建一个仓库(repository) 来存储之后的博客网站，点击首页任意位置出现的 New repository按钮创建仓库, Respository name 中的username.github.io 的username 一定与前面的Owner 一致，记住你的username下面会用到。

安装工具

1.安装git

windows10官网下载：https://git-scm.com/download/win

linux: $ brew install git // 安装Git

2.安装Nodejs

windows去nodejs官网下载： http://nodejs.cn/download/

所有必须工具已经安装完成 。

创建、配置博客

创建本地博客

将下面的 username 替换成你自己的username，执行成功后，会创建出一个名为 username.github.io 的文件夹,之后所有的操作都是在这个文件夹中进行的。

hexo init username.github.io

npm i hexo-server hexo 3.0把服务器独立成个别模块，需要单独安装

配置博客

基础配置：打开文件位置username.github.io/_config.yml修改几个键值对，下面把几个必须设置的列出来按需求修改 ：

注意配置的键值之间一定要有空格

配置github账户信息

回到gitbash中，配置github账户信息（YourName和YourEail都替换成自己的）：

git config --global user.name "YourName"

git config --global user.email"YourEail"

创建SSH
在gitbash中输入：ssh-keygen -t rsa -C "youremail@example.com，生成ssh。然后按以下的方式找到id_rsa.pub文件的内容。

cd ~/.ssh

cat id_rsa.pub

之后将产生的ssh-rsa复制，打开github进入personal settings->SSH and GPA keys ->new SSH key

打开新建窗口后，ssh名随意填，内容将刚刚本地产生的ssh密钥添入并保存。

回到本地gitbash中，输入 ssh -T git@github.com 测试gitbash中的密钥验证是否添加成功。

测试本地访问

hexo generate 根据配置生成html文件以供浏览器访问

hexo server 启动服务器，用浏览器访问 http://localhost:4000 测试

同步至github

*安装hexo-deployer-git自动部署发布工具 *

npm install hexo-deployer-git --save

同步上传

hexo clean && hexo g && hexo d

访问

博客已经完成了，在浏览器中输入 https://joke66.github.io/ 就能够访问了。
